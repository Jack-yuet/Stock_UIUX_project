# 股票评分系统重构（V2 数据驱动版）

本文档描述当前已上线的“去冗余 + 门控 + 市场状态自适应”的评分机制，并给出下一阶段“大幅精细化 + 回测驱动 + 概率校准”的实施方案。保持前后端接口不变（finalScore、breakdown、details），仅优化计算内核与流程。

## 1. 现行评分结构（已实现）
- 总体：trendScore(0~33) + technicalScore(0~35) + consistencyBonus(0~3) + patternScore(0~10) + marketEnvironmentScore(-9~+9) + trendPositionScore(0~7) → 再乘以 volumeModifier(0.9~1.1)，标准化至 0~10。
- 去冗余：技术只保留 MACD、RSI、布林带、变化率、OBV 五类正交核心。
- 门控：形态分仅在“与趋势同向 + 放量”计分；市场状态对内部得分做缩放（熊：技术×0.8、形态×0.5；震荡：趋势×0.8、形态×0.7；牛：技术×1.05）。
- 阈值：买入≥3.5；卖出<2.0；中性其它。

## 2. 信号定义与量化准则
- 趋势（33）：
  - MA20vsMA50（看涨/看跌 → ±0.8×0.45）
  - MA 多头/空头排列（±0.8×0.25）
  - ADX 趋势强度（强/中/弱 → 1.0/0.5/0 ×0.15）
  - HT_TRENDMODE（趋势/无趋势 → +1.0/-0.5 ×0.15）
  - 合成后 clamp[-1,1] × 33
- 技术（35）+ 一致性（≤3）：
  - 指标强度映射至 [-1,1]，求均值×35；≥70%同向 → +(ratio-0.7)×10，cap 3
- 形态（≤10，强门控）：
  - 仅当“与趋势方向一致 + 量比>1.3”全额计分；仅一致不放量 → 0.5×；否则 0
- 市场环境（-9~+9，且缩放门控）：
  - bull/sideways/bear 基于 factor/regime 给对称正负分，并对内部得分缩放
- 趋势位置（0~7）：
  - HT_DCPHASE + 区间位置/均线偏离度综合映射
- 成交量修正（0.9~1.1）：
  - volumeModifier = 1 + (量比-1)×0.2，clamp 至 [0.9,1.1]

## 3. 大幅精细化路线（不改接口）

### 3.1 特征收敛与去冗余
- 家族化：趋势/动量/波动/形态/量能/位置；每家族保留1-2个正交代表。
- 分位数标准化：所有特征→分位得分[-1,1]，消除量纲影响。
- 联动扣分：高波动、低流动性、事件日（财报/停牌）降低贡献。

### 3.2 形态“触发器化”
- 形态不再直接拉高总分，而是作为“触发器”：当趋势/技术满足阈值，形态+放量+关键位才给予少量加分（上限 10）。
- 将形态列表精简为统计显著的一小组；罕见/噪声形态剔除。

### 3.3 市场状态自适应（门控再细化）
- regime 识别增强：
  - 趋势：大盘 MA200 斜率/ADX
  - 波动：ATR/布林宽度
  - 广度：上涨家数比例、行业扩散度
- 缩放矩阵按 regime/风格动态更新（回测驱动）。

### 3.4 概率化评分与阈值（核心）
- 训练目标：未来 5/10/20 日超额收益的“上涨概率 p”或“期望收益 E[r]”。
- 轻量模型：Logistic/XGBoost/LightGBM（避免过度复杂）。
- 特征：采用第 3.1 的正交特征，外加 regime 与流动性因子。
- 校准：Platt/Isotonic Calibration，使 p 与真实胜率单调一致。
- 融合：finalScore 由“规则得分 S”和“概率 p/E[r]”线性融合：
  - finalScore' = α·S + (1-α)·f(p/E[r])，α 由回测寻优；f 为线性或单调映射至 0-10。
- 阈值：买入阈值与卖出阈值由收益曲线/胜率曲线分层回测确定（而非拍脑袋）。

## 4. 回测框架（落地执行）
- 数据构造：
  - 样本：滚动窗口生成（2000+ 交易日），训练集→验证→测试（时间走步）。
  - 标签：forward return（扣成本），或 1/0（正向/反向）。
- 策略规则：
  - 选股：每日按 finalScore' 排序取 Top-N，或 p≥p_buy；行业权重/单票权重上限。
  - 开平仓：买入≥阈值；卖出<阈值或时间止损（如10/20日）。
  - 仓位：与ATR/波动倒数成比例；大盘风险偏高时整体降杠杆。
  - 交易成本：按双边费率+滑点扣减。
- 评价指标：年化收益、Sharpe、最大回撤、信息比、Turnover、命中率、IC/RankIC、分层收益曲线。
- 稳健性：不同年份/行业/市值子样本，和 bull/bear/sideways 阶段分别评估。

## 5. 监控与在线再校准
- 上线后记录（score/概率）与实际胜率的分桶曲线，定期做小幅再校准（避免频繁重构）。
- 观察 regime 切换期的错误类型（假阳/假阴），微调缩放矩阵与 α。

## 6. 数据结构与兼容
- breakdown 字段保持：
  - trendScore, technicalScore, patternScore, marketEnvironmentScore, trendPositionScore, consistencyBonus, volumeModifier
- details 建议扩展：
  - technicalSignals（各核心指标的 strength）
  - marketEnvironment（factor、regime、广度/波动快照）
  - modelProb（若启用ML：p 与校准后的 p_calibrated）
  - regimeScales（各分项缩放系数快照）

## 7. 阈值与执行建议（运行参数）
- 阈值：买入 ≥ 3.5；卖出 < 2.0（建议在 bear 上调买入阈值至 3.8~4.2，或减少持仓数量）。
- Top-N 与分散：控制行业/单票权重，降低拥挤与回撤。
- 时间止损：10/20 日无效退出；ATR 止损与跟踪止盈结合。

---

结论：
- 先用现行“规则+门控+状态缩放”的 V2 版本稳住噪声与回撤；
- 并行构建“回测+概率校准”的离线流程，用数据驱动 α、阈值与缩放矩阵；
- 待稳定后，将 finalScore 与概率融合上线（接口不变），实现真正“概率化”的评分与交易信号。
